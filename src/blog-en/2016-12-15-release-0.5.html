Domogik 0.5 release

The Domogik team is proud to announce the release of Domogik 0.5!

This release is to be used with Domoweb 0.5 (the web interface) and the Domogik-MQ 1.4 component.

You can check the improvments list in the release note.

Domogik

This Domogik release include several big technical improvments, mainly on architecture and performances. The plugins development process has also been improved by allowing the development of non xPL plugins.

A new scenario engine have been implemented.

A new timeline section allows to display the timeline for all Domogik, a plugin or a device.

Another new section have been added to show the devices batteries levels.

There were many other improvments : new data types, backup script, docker image (for test only), SSL support, users managements, metrics, ...

Packages

The official packages list is available on our packages repository. Today (december), this list is quite short because lot of the existing plugins have not been tested yet. There is currently amore than 40 plugins in development. Most of them are already working fine.

Domoweb, the web interface includes also a lot of improvments :

Domodroid

The new Domodroid release includes also a lot of improvments :

  • The configuration menu have been rewrite.
  • The configuration can now be done by scanning a QR-code available from the Domogik administration home page.
  • Map mode improvment.
  • New widgets.
  • Improvment of existing widgets.
  • ...

Documentation

The end user documentation have been reviewed on visual and content. It is now more clear.

To conclude...

This was a big release of Domogik. We want to thanks everybody involved in the project : developpers, testers, issues creators, ...

The next release will be a technical one, focused on the installation and resources improvments.
